# 開発引き継ぎまとめ

作成日: 2026-05-10

## プロジェクト概要

このプロジェクトは当初、毎日のタスクや長期目標からAIが時間割を作る「万能型AIスケジュール管理 / AI秘書」アプリとして開始した。

その後、MVPとしてはスコープが広すぎるため、第一段階では「DTMer向け制作継続支援アプリ」に方向修正した。

長期ビジョンは変わらず、将来的には「人の目標達成を支援する万能型AI」へ横展開する。ただし現段階では、時間割生成よりも以下を重視する。

- 進捗理解
- 制作継続
- 小さい完成体験
- 次にやるべきこと提案

この方針は `project_overview.md` にまとめてあり、今後の開発方針として扱う。

## 現在のアプリの方向性

現在のMVPは「DTM進捗管理AI / 制作継続支援アプリ」。

目的は、DTMerが曲を途中で諦めず完成しやすくすること。

現在のトップ画面は、管理画面ではなく「毎日スマホで開いて、次にやることがすぐ分かる」体験を優先している。

画面の優先順位は以下。

1. 今日やること
2. 完成率
3. 次工程
4. 進捗ログ入力
5. 制作工程一覧
6. AI提案詳細
7. 過去ログ

スマホファーストの1カラム構成で、PCでも中央寄せの自然なカード配置にしている。

## 実装済み機能

### 曲管理

曲を登録できる。

保存項目:

- 曲名
- ジャンル
- 目標完成日
- メモ
- 現在状態

現在状態:

- idea
- writing
- arranging
- mixing
- mastering
- posted
- paused

### 制作工程管理

曲を登録すると、以下の初期工程が自動作成される。

- アイデア
- コード
- メロディ
- Aメロ
- Bメロ
- サビ
- 編曲
- ミックス
- マスター
- 投稿

各工程の状態:

- not_started
- in_progress
- done

工程は1つずつ更新する方式から、すべての工程を入力後に「工程をまとめて更新」で一括保存する方式へ変更済み。

### 完成率表示

工程の `done` 数から完成率を計算して表示する。

例:

```text
3 / 10 完了 -> 30%
```

### 進捗ログ

曲ごとに進捗ログを保存できる。

保存項目:

- 日付
- 作業時間
- やったこと
- 詰まったこと
- 次にやりたいこと
- 自己評価 1〜5

最新ログは初期表示で見える。過去ログ全体は折りたたみで見られる。

### AI提案

AI提案欄を実装済み。

OPENAI_API_KEY がある場合は OpenAI API を使う。未設定の場合はローカル簡易提案で動く。

提案内容:

- 次にやるべき作業
- 小さい完成目標
- 詰まりへの軽いアドバイス
- 次回作業候補

OpenAI APIでは Responses API の Structured Outputs を使う方針。

### モバイルファーストUI

現在の初期表示:

- 選択中の曲名
- 完成率
- 今日やること / 次にやること
- 次工程
- 進捗ログを追加するボタン

折りたたみ化済み:

- 進捗ログ入力
- 制作工程一覧
- AI提案の詳細
- 過去ログ
- 曲切り替え・曲追加
- 詳細メモ

PC表示では以前、2カラム指定によって左側に大きな空白が出ていた。原因は `.mobile-stack` のPC向け2カラム指定と `grid-column` の左右固定だったため、現在はPCでも中央寄せの1カラム構成に修正済み。

## 既存機能として残っているもの

当初作った時間割生成系のコードやDBテーブルは完全削除していない。

ただし現在のトップ画面ではメイン機能としては出していない。

残っている主な旧機能:

- 今日やることから時間割を作るローカルスケジューラ
- 長期目標から作業候補を作る簡易ロジック
- 日次タスク / daily_plans / goal_progress_logs などの旧テーブル

今後、必要に応じて整理またはDTM向け機能へ統合する。

## 直近の重要な修正

### 制作工程の一括更新

以前は工程ごとに個別の更新ボタンがあった。

現在は、すべての工程ステータスを選んだあと、1回の「工程をまとめて更新」で保存する。

実装:

- `updateSongStepsAction`
- `updateSongSteps`
- `StepBoard` を単一フォーム化

### PC表示の大きな空白修正

以前はPC幅で2カラム化していたため、左側に大きな空白が発生した。

現在は以下の方針に修正済み。

- PCでも1カラム
- 最大幅を抑えて中央寄せ
- スマホと同じ順序で縦並び
- `.mobile-stack` の2カラム指定と `grid-column` 固定を削除

## 今後の開発方針

優先すること:

1. 曲登録
2. 工程更新
3. 進捗保存
4. AI提案
5. 小さい完成目標の精度向上
6. 進捗ログから詰まりや傾向を分析

今はやらないこと:

- 自動作曲
- MIDI生成
- DAW連携
- 高度なミキシング解析
- SNS
- 人生全体管理
- 複雑な時間割生成

## 次にやるとよさそうなこと

候補:

   - 曲名、ジャンル、目標完成日、メモ、現在状態を後から編集できるようにする。

4. 小さい完成目標のテンプレート化

   - 「サビのコードだけ」
   - 「仮音色で全体を並べる」
   - 「ミックス前に構成を固定」

5. README更新
   - 現在のDTM MVP仕様に合わせてREADMEを書き直す。

## 注意点

- 既存コードを全削除しない方針。
- ただし、現在のMVPでは旧スケジュール機能は主役ではない。
- UIはスマホファーストを維持する。
- 「今日やること」は常に見えるようにする。
- 詳細情報は折りたたみに逃がす。
- 管理画面ではなく、毎日スマホで開きたくなる体験を重視する。
